**Date:** 2026-06-14
**Status:** Approved (brainstorm complete)
**Milestone:** v1.0-M2 (second of three milestones to the v1.0 acceptance bar)
**Parent bar:** `docs/superpowers/specs/2026-06-14-v1.0-acceptance-bar-design.md`
**Depends on:** v1.0-M1 (self-service onboarding — merged `acc725b`)

## Goal

Make **one `docker compose up`** at `deploy/compose/` honestly bring up **all six
AgentCore pillars healthy** on a clean single host, with **semantic Memory
actually working out of the box** (bundled local embedder + auto-created `vector`
extension), identity **on** with a one-command-generated bootstrap, and
**documented persistence/reset**.

## Reframe (what changed since the bar was written)

The acceptance bar predicted M2's headline gap as "swap the bundled Postgres for
a pgvector-capable image." **That gap is already closed** — every existing
compose file (`deploy/docker-compose.yml`, `.full.yml`) already uses
`image: pgvector/pgvector:pg16`. The *real* M2 gaps, found by inspecting the
current `deploy/` tree, are:

1. **No single "all six pillars" compose.** `docker-compose.yml` brings up only
   Postgres; `.full.yml` adds runtimed but **not** sandboxes, browser, or the
   four observability services (those live in `docker-compose.obs.yml`). "One
   `docker compose up` → all six pillars" is not true today.
2. **`CREATE EXTENSION vector` is still manual.** The image *ships* pgvector, but
   `internal/memory/embed_schema.sql` documents that an operator must run
   `CREATE EXTENSION` once (the unprivileged `runtime` role can't — it needs a
   superuser). Not automated.
3. **No turnkey embedding provider.** Semantic recall needs `RUNTIME_EMBED_*`
   pointing at an OpenAI-compatible `/embeddings`. A stranger has no API key — so
   "semantic recall works out of the box" needs a bundled, air-gap answer.
4. **Sandbox/browser need Docker access** when runtimed is itself a container
   (socket reach + sibling images present on the engine + `host.docker.internal`).
5. **Identity is off in the demo compose.** `.full.yml` deliberately runs **open**
   (unauthenticated), but the M1 onboarding UI + tenant-isolation proof steps
   require identity **on** with a superuser bootstrap.
6. **Persistence/reset unspecified** for a real self-host story.

So M2 is: **"make one compose honestly bring up all six pillars, with semantic
Memory actually working."** This stays strictly within the frozen bar (zero new
pillar capability).

## Design Decisions (settled in brainstorm)

| # | Decision | Rationale |
|---|---|---|
| 1 | **Bundled local embedder** (custom FastEmbed/ONNX container) | Air-gap friendly, matches the project's stated on-prem values; proof passes with zero external accounts. Model baked at **build** time ⇒ no first-run download. |
| 2 | **Mount host `/var/run/docker.sock`** into runtimed | Standard, simple, works on Linux + Docker Desktop; the code already maps loopback→`host.docker.internal`. Documented single-node root-equivalent trust tradeoff. |
| 3 | **Sidecar images via compose `build` + a no-up `build-only` profile** | `docker compose build` produces `runtime-sandbox:latest` / `runtime-browser:latest` on the engine without running them; preserves the one-command promise. |
| 4 | **One turnkey file at `deploy/compose/`** is the v1.0 surface | `docker compose up` brings up all six pillars. Existing `deploy/docker-compose*.yml` stay as-is for contributors. |
| 5 | **Identity ON by default; bootstrap via `.env`** | The M1 onboarding UI + tenant isolation + cross-tenant-refused steps require identity on. Console login works on first boot. |
| 6 | **`make compose-init` generates `.env`** | A script generates a strong bootstrap key + AES key into `deploy/compose/.env` (gitignored). Lowest stranger friction; no hand-rolled crypto; no weak placeholder reaching a running stack. |
| 7 | **Named volumes + documented reset** | Postgres (and embedder model cache, if any) on named volumes; data survives `down`; reset = `down -v`. Matches the bar's "documented persistence/reset." |

## Architecture & Component Boundaries

A new self-contained directory `deploy/compose/` is the v1.0 surface. Existing
`deploy/docker-compose*.yml` are **untouched** (contributor/dev workflows).

```
deploy/compose/
  docker-compose.yml        # THE turnkey file — all six pillars
  .env.example              # documented vars (committed)
  .env                      # generated by `make compose-init` (gitignored)
  init.sh                   # secret generator (bootstrap key + AES key) → .env
  embedder/
    Dockerfile              # FastAPI + fastembed ONNX; model baked at build
    app.py                  # OpenAI-compatible POST /embeddings + GET /healthz
    requirements.txt
    test_app.py             # determinism + dim unit test
  initdb/
    01-create-extension.sql # CREATE EXTENSION IF NOT EXISTS vector;
  runtime.compose.yaml      # agent registry for the turnkey stack (identity-on)
  smoke.sh                  # M2 capstone gate — brings up the stack, asserts 6 pillars
```

### Services in `docker-compose.yml`

| Service | Image / build | Pillar(s) | Notes |
|---|---|---|---|
| `postgres` | `pgvector/pgvector:pg16` | Memory, Identity, Runtime | mounts `initdb/` → auto `CREATE EXTENSION vector`; named volume `pgdata`; healthcheck `pg_isready -U runtime` |
| `embedder` | build `embedder/` | Memory | OpenAI-compatible `/embeddings`; model baked at build (air-gap); healthcheck `GET /healthz`; dim 384 |
| `runtimed` | build root `deploy/Dockerfile` | Runtime, Gateway, Identity, Sandboxes, Observability-source | identity on; mounts `/var/run/docker.sock`; `host.docker.internal` extra_host; `depends_on` postgres + embedder healthy; mounts `runtime.compose.yaml` |
| `prometheus` | `prom/prometheus:latest` | Observability | scrapes runtimed `/metrics` (reuses `deploy/prometheus.yml` pattern) |
| `grafana` | `grafana/grafana:latest` | Observability | provisioned dashboard + datasource (reuses `deploy/grafana/`) |
| `otel-collector` | `otel/opentelemetry-collector-contrib:latest` | Observability | OTLP/HTTP :4318 → jaeger (reuses `deploy/otel/collector-config.yaml`) |
| `jaeger` | `jaegertracing/all-in-one:latest` | Observability | traces UI :16686 |
| `sandbox` | build `../sandbox.Dockerfile`, **profile `build-only`** | Sandboxes | tags `runtime-sandbox:latest`; never `up` |
| `browser` | build `../browser.Dockerfile`, **profile `build-only`** | Sandboxes | tags `runtime-browser:latest`; never `up` |

`docker compose build` builds every service including the two profiled ones (so
the sibling images land on the host engine). `docker compose up` starts the
seven long-running services; runtimed launches sandbox/browser siblings on
demand against the mounted socket.

> **Profiles note:** the `build-only` services carry `profiles: ["build-only"]`
> so they are part of `docker compose build` but excluded from a default
> `docker compose up`. Confirm during implementation that the installed compose
> version builds profiled services on a bare `docker compose build`; if not, the
> fallback is `docker compose --profile build-only build` documented as the one
> build command. Either way the **run** command stays a bare `docker compose up`.

### Wiring details that make it actually work

1. **`CREATE EXTENSION` automation** — the pgvector image runs `*.sql` in
   `/docker-entrypoint-initdb.d/` as the superuser on **first init only**.
   `initdb/01-create-extension.sql` (`CREATE EXTENSION IF NOT EXISTS vector;`)
   closes the unprivileged-role gap with zero operator action. First-init-only is
   acceptable: the named volume persists; a documented `down -v` reset re-runs it.

2. **Embedder ↔ Memory contract** — runtimed env:
   `RUNTIME_EMBED_MODEL=bge-small-en-v1.5`, `RUNTIME_EMBED_DIM=384`,
   `OPENAI_BASE_URL=http://embedder:8000`, `OPENAI_API_KEY=` (empty; the shim
   ignores auth). The existing `httpEmbedder` (`POST {base}/embeddings`,
   `internal/memory/embed.go`) works unmodified.
   - **Recall-floor recalibration (load-bearing).** `RUNTIME_EMBED_RECALL_FLOOR`
     defaults to `0.25`, tuned for OpenAI `text-embedding-3-small`
     (`internal/agentkind/registry.go:105`). BGE-small's cosine-similarity range
     differs; a wrong floor makes recall **silently return nothing**, which would
     fail the M3 proof invisibly. M2 sets `RUNTIME_EMBED_RECALL_FLOOR` in compose
     to a value **empirically measured against the bundled model**, and `smoke.sh`
     asserts a non-lexical recall lands above the floor.

3. **Sandbox/browser via socket** — `runtimed` gets
   `volumes: ["/var/run/docker.sock:/var/run/docker.sock"]` +
   `extra_hosts: ["host.docker.internal:host-gateway"]`. The code already maps a
   loopback CDP host → `host.docker.internal` (`internal/browser/docker.go:76`)
   and connects via `client.FromEnv` (default socket) in both
   `internal/sandbox/docker.go:56` and `internal/browser/docker.go:54`. **No Go
   changes expected**; if the containerized launch surfaces a host-mapping or
   uid/path gap, fix it minimally under this milestone.

4. **Identity bootstrap** — `runtime.compose.yaml` ships agents with identity
   active; `.env` supplies `RUNTIME_ADMIN_BOOTSTRAP` (superuser break-glass key),
   `RUNTIME_SECRETS_KEYS="proofkey:<base64>"`, and
   `RUNTIME_SECRETS_PRIMARY=proofkey`. (The `_PRIMARY` must name the key id in
   `_KEYS` — a live-proof bug from M1; encoded here so the generator gets it
   right.) Console `/ui/onboarding` (M1) is reachable on first boot; the
   superuser creates the tenant + admin per the proof.

### `make compose-init` / `init.sh`

Generates a strong bootstrap key (`openssl rand -hex 32`) and an AES-256 key
(`openssl rand -base64 32`), writes `deploy/compose/.env` from `.env.example`
with the values filled and `RUNTIME_SECRETS_PRIMARY` matching the generated key
id. Idempotent-safe: **refuses to clobber an existing `.env`** unless `--force`.
No hand-rolled crypto; no weak placeholder reaching a running stack. A `Makefile`
target `compose-init` wraps it.

### Boundary check

The embedder knows nothing about runtime (just HTTP). runtimed reaches
Postgres/embedder by **service name** over the compose network. The socket mount
is the only host coupling, isolated to one service and documented. Each piece is
independently testable: embedder via `curl /embeddings`; extension via `\dx`;
`init.sh` against a temp dir; the whole stack via `smoke.sh`.

## Embedder Image (detail)

- **Base:** `python:3.12-slim`. **Deps:** `fastapi`, `uvicorn`, `fastembed`.
- **Model:** `BAAI/bge-small-en-v1.5` (dim **384**), **downloaded at image build
  time** into the image layer (a `RUN python -c "from fastembed import
  TextEmbedding; TextEmbedding('BAAI/bge-small-en-v1.5')"` warms the cache) so the
  running container needs **no network** — true air-gap after `compose build`.
- **API:** `POST /embeddings` accepting `{"model": "...", "input": "<text>"}` and
  returning `{"data": [{"embedding": [..384 floats..]}]}` — the exact shape
  `httpEmbedder` decodes. `model` is accepted and ignored (single bundled model).
  `GET /healthz` → `200 {"status":"ok"}` for the compose healthcheck.
- **Determinism:** same input ⇒ identical vector (asserted in `test_app.py`).

## Testing

### Unit / hermetic (fast, no full stack)

- **`init.sh`** (bash test): run against a temp `HOME`/dir → asserts `.env`
  written with a 64-hex bootstrap key, a valid `id:base64` secrets key, and
  `RUNTIME_SECRETS_PRIMARY` equal to that id; re-run without `--force` → refuses
  (non-zero exit, `.env` byte-identical); `--force` → regenerates new values.
- **embedder `test_app.py`** (`pytest`): post `{"model":"x","input":"hello"}` to
  the app → `len(data[0].embedding) == 384`; same input twice → identical vector;
  `GET /healthz` → 200. Runs in CI via the embedder build stage or host Python.

### The M2 smoke check — `deploy/compose/smoke.sh`

One script; runs the real stack; asserts every pillar's substrate is live. **This
is M2's capstone gate** (M3 wraps it in docs-driven prose + cross-tenant/agent
depth). Steps:

1. `docker compose build` (incl. `build-only` profile ⇒ sandbox/browser images
   exist) → `docker compose up -d`.
2. Wait for health (poll with timeout): postgres, embedder, runtimed, prometheus,
   grafana, otel-collector, jaeger.
3. **Memory/pgvector** — `psql ... -c '\dx'` shows `vector`; **embedder** —
   `curl :<embport>/embeddings` returns a 384-vector.
4. **Recall-floor proof** — write a memory fact via the API; semantically recall
   it with a query that does **not** lexically match; assert it returns **above
   the configured floor**. (Catches a mis-tuned `RECALL_FLOOR` for BGE-small.)
5. **Runtime + Identity** — with the bootstrap key, an admin route returns 200;
   an unauth request returns 401. Bootstrap → create tenant → mint key (proves
   the M1 onboarding API is reachable on this stack).
6. **Gateway** — register the `examples/rest-demo` OpenAPI upstream via the
   onboarding API; assert it reaches `state=up`.
7. **Sandboxes** — drive the interpreter tool so a sandbox sibling launches and
   returns computed output (proves docker.sock sibling-launch); **browser** — a
   health/CDP probe that the browser sibling launches.
8. **Observability** — `curl prometheus/api/v1/targets` shows runtimed `up`;
   Jaeger has ≥1 trace after the turn; Grafana `/api/health` → 200.
9. **Persistence** — `docker compose down` (no `-v`) → `up` → the step-4 fact
   survives. Then `down -v` → volume gone (documented reset).
10. **No secrets in logs** — grep captured container logs for the bootstrap/AES
    values → must be **absent** (standing project rule).
11. Teardown clean.

`smoke.sh` lives in-repo and is re-runnable, so M3 can call it as the inner loop
of `v1-proof.sh`.

### Live proof (M2)

Per every prior milestone: after all tasks, a **holistic review**, then
`smoke.sh` must **pass clean from a fresh `docker compose build` on this
machine** before merge. M2 proves the substrate works; M3 proves a *stranger*
can do it from docs alone.

## Definition of Done

- [ ] `deploy/compose/docker-compose.yml` brings up all six pillars; a bare
      `docker compose up` runs the seven long-running services.
- [ ] `docker compose build` produces `runtime-sandbox:latest` +
      `runtime-browser:latest` via the `build-only` profile (or one documented
      `--profile build-only build` if the compose version requires it).
- [ ] Bundled `embedder/` image: OpenAI-compatible `/embeddings`, model baked at
      build (air-gap), `/healthz`, dim 384.
- [ ] `initdb/01-create-extension.sql` auto-creates the `vector` extension on
      first init.
- [ ] `RUNTIME_EMBED_RECALL_FLOOR` empirically tuned for the bundled model;
      `smoke.sh` asserts non-lexical recall above the floor.
- [ ] `runtimed` reaches Docker via the mounted socket; sandbox + browser
      siblings launch; `host.docker.internal` wired.
- [ ] Identity on by default; `make compose-init` generates `.env` (bootstrap +
      AES key, `_PRIMARY` matching the key id); console onboarding reachable on
      first boot.
- [ ] Named volumes; `down` preserves, `down -v` resets — documented in
      `.env.example` / compose comments.
- [ ] `init.sh` + embedder unit tests green; `go build` / `go vet` / `gofmt`
      clean (any Go touched).
- [ ] No secrets / secret-names in logs (asserted in `smoke.sh`).
- [ ] `smoke.sh` passes clean from a fresh build; holistic review done; merged to
      `master`.
