## Design Decisions (settled in brainstorm)

| # | Decision | Rationale |
|---|---|---|
| 1 | **Deterministic direct-call proof** (no LLM key) | The bar's "real agent turn" needs an API key, conflicting with M2's air-gap/zero-account choice. The e2e tests already prove every pillar via direct MCP/HTTP calls. "All six pillars demonstrably exercised" is the gate, not the literal word "agent". |
| 2 | **`v1-proof.sh` supersedes `smoke.sh`** | One canonical gate at the bar-named path `deploy/compose/v1-proof.sh`. M2's `smoke.sh` already does the substrate checks; promote + deepen it (gateway tool-CALL, sandbox execute_code, Jaeger trace, cross-tenant refusal). Removes duplication. |
| 3 | **Three new `docs/` files + README links** | The bar names three guides; keep them discoverable. The existing 1955-line dev-focused README stays for contributors; a short "v1.0 turnkey" section links the guides. |
| 4 | **Clean-as-possible proof on this host** | Fresh `git clone` of both repos into a temp dir, `docker compose build --no-cache`, run only commands copy-pasted from the docs, no author env. Honest within the macOS/Docker-Desktop constraint (residual clean-Linux gap noted as a caveat). |
| 5 | **Create `v1.0` tag locally, do not push** | Matches the "commit/push only when asked" convention while completing the milestone deliverable. The user reviews + pushes. |
| 6 | **Tiny Go probe for MCP-session calls** | `/gateway/mcp` is Streamable-HTTP MCP needing a session handshake — not curl-able. A small `cmd/v1-probe` does the gateway tool-call + sandbox execute_code; bash handles the curl-able assertions. |

## Architecture

### The harness-sibling reality (stated in quickstart)

`runtime`'s `go.mod` has `replace github.com/sausheong/harness => ../harness`,
and `deploy/Dockerfile` `COPY harness/ ./harness/`. A stranger must clone **both
repos side-by-side**. Vendoring harness is a bigger v1.x change — out of scope;
the quickstart states the two-repo clone up front.

### `cmd/v1-probe/` (new, ~150 lines)

A single-purpose CLI that opens an authed MCP session to `/gateway/mcp` (reusing
the client pattern from `test/gateway_sandbox_e2e_test.go`: `sdk.NewClient` +
`StreamableClientTransport` with a bearer round-tripper, MCP SDK v1.5.0) and runs
subcommands the proof invokes:

- `v1-probe list --base <url> --key <bearer>` — prints federated tool names;
  exits non-zero unless both `orders__listOrders` (REST adapter) and
  `sandbox__execute_code` (MCP upstream) are present. Polls with fresh sessions
  until federated (stdio upstream dial is async) — mirrors `connectWhenFederated`.
- `v1-probe call-rest --base --key` — calls `orders__listOrders` through the
  gateway; asserts a non-empty orders result. Proves the **Gateway REST-adapter**
  call end-to-end.
- `v1-probe sandbox --base --key` — `sandbox__create_sandbox` then
  `sandbox__execute_code` with `print(6*7)`; asserts `42` in the output. Proves
  **Sandboxes** code execution via the mounted docker.sock sibling-launch.

Built in the runtimed image (added to the Dockerfile build list) **and**
`go run ./cmd/v1-probe`-able from the host (the proof runs it from the host
against `localhost:8080`).

### `deploy/compose/v1-proof.sh` (supersedes smoke.sh) — pillar-by-pillar gate

Builds on smoke.sh's structure (init → build → up → assert → teardown), with one
explicit labeled assertion per pillar:

| # | Pillar | Assertion |
|---|---|---|
| 1 | **Runtime** | stack up; `/healthz` 200; `runtimectl invoke` round-trips a session (scripted agent → "final answer") |
| 2 | **Identity** | unauth `/admin/upstreams` → 401; bootstrap creates tenant + mints an agent key; **cross-tenant refused** (tenant-B's key sees no tenant-A upstream on `/gateway/status`) |
| 3 | **Gateway** | self-service register `orders` OpenAPI upstream → `state=up`; `v1-probe list` shows `orders__*` AND `sandbox__*`; `v1-probe call-rest` returns orders ⇒ an **MCP upstream and a REST upstream both federated** (bar step 3) |
| 4 | **Memory** | write a fact + non-lexical semantic recall clears the calibrated floor (direct embedder cosine) — proves pgvector-in-compose |
| 5 | **Sandboxes** | `v1-probe sandbox` → `execute_code` returns `42` (real container via docker.sock); browser image present on engine |
| 6 | **Observability** | Prometheus `runtimed` target up + a `runtime_*` metric present; **Jaeger has a trace** (`GET :16686/api/traces?service=runtimed` ≥1 after a turn, or `/api/services` lists `runtimed`); Grafana `/api/health` 200 |

Plus standing gates: **per-tenant credential at dial** (register `orders` with a
`cred_secret`/`cred_header` pointing at a tenant secret; rest-demo accepts any
header so it stays deterministic — proves the inject-at-dial path on compose);
**persistence** (row survives `down`→`up`); **no secret value/name** in
`docker compose logs`; clean teardown (incl. the rest-demo reap fix from smoke.sh).

### Docs-only discipline

Every command the proof runs must appear **verbatim** in one of the three guides.
The proof is the runbook made executable; a step not in the docs is a doc bug
(the bar's forcing function). Cross-checked during the capstone run.

## Docs Structure

### `docs/quickstart.md` (short — one screenful to "it's up")
- Prereqs: Docker; **clone both `runtime` and `harness` side-by-side** (the
  `replace`/Dockerfile sibling requirement, stated first).
- Commands: `cd runtime/deploy/compose` → `make compose-init` →
  `docker compose build` → `docker compose up`.
- "What you should see": seven services healthy; console `:8080/ui`, Grafana
  `:3000`, Jaeger `:16686`.
- Pointers to operator guide (login) + tenant guide (onboard).

### `docs/operator-guide.md`
- Bootstrap: `.env` `RUNTIME_ADMIN_BOOTSTRAP` (generated by `make compose-init`);
  console login with it.
- Ports table; `DOCKER_GID` on native Linux (`getent group docker`); the
  **docker.sock root-equivalent trust posture** (single-node only).
- Persistence/reset: `down` preserves, `down -v` resets + re-runs pgvector init.
- Observability URLs (Grafana/Jaeger/Prometheus) + what to look for.
- Security caveats consolidated (no secrets in logs; not for untrusted networks).

### `docs/tenant-guide.md`
- Onboard **through the console UI**: create tenant → mint agent key → set a
  credential → register the OpenAPI upstream (`examples/rest-demo`) → see `up`.
- How each pillar is exercised (the deterministic commands, mirroring the proof).
- Optional, signposted: "point an agent at a real LLM" (deferred to v1.1, noted).

### README
A short "## v1.0 — turnkey self-host" section linking the three guides; the
existing dev Quick start stays for contributors.

## Testing

### Unit / hermetic
- `cmd/v1-probe` compiles (`go build ./cmd/v1-probe`); `go vet`, gofmt clean.
- (Probe behavior is validated in the live proof against the real stack — a
  hermetic test would need a full MCP gateway fake; the proof is the real check.)

### The capstone proof (M3's gate — clean-as-possible)
1. Fresh `git clone` of **both** `runtime` and `harness` into a temp dir.
2. `docker compose build --no-cache` (nothing pre-built) per quickstart verbatim.
3. `docker compose up`; run **only** commands copy-pasted from the three guides.
4. `v1-proof.sh` exercises all six pillars per the table + standing gates.
5. Iterate docs/script until green. Residual macOS-vs-Linux gap noted as a caveat.
6. Teardown clean.

**Discipline (every prior milestone):** holistic review after all tasks, then the
proof passes clean before merge.
